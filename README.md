# [ ~ 05.09]

## 1. MMS 이미지 전송 안정성 확보
- 기존 파일 경로 방식에서 벗어나, 앱 내부에서 이미지를 즉시 압축하고 포맷을 완벽한 `JPEG(300KB 이하)`로 강제 변환하여 솔라피 서버로 전송하도록 아키텍처 개편.
- MMS 발송 시 자동으로 메시지 내용이 제목으로 복사되어 두 번 전송되는 현상을 막기 위해, 제목을 투명 공백으로 고정 처리.

## 2. 문자 발송 내역(이력) 및 UI 고도화
- 이력 모달에서 차감된 잔액 및 건수가 `[object Object]`로 표시되던 오류 수정 및 천 단위 콤마(`,`) 적용 완료.
- 이미지(MMS)를 첨부해서 보냈을 경우, 발송 이력 모달에서도 모바일 썸네일처럼 첨부된 이미지를 직접 확인할 수 있도록 개선 (Base64 로컬 저장).

## 3. 문자 연동 가이드 및 매크로 변수 동적 지원
- 사용자가 직접 엑셀 파일 열 이름을 몰라도 클릭만으로 치환 문자를 삽입할 수 있도록, 엑셀 데이터 기반의 `동적 치환 변수 뱃지` UI 추가 (일반 발송 및 자동 발송 모달 모두 적용).
- 초보자를 위한 솔라피 연동 가이드 창 제작 (가입 -> 번호 등록 -> API 키 생성 과정 상세 안내).

## 4. 엑셀 중복 판단 조건 강화
- 엑셀 업로드 시 기존에는 '신도번호' 하나만으로 중복을 판단했으나, `신도번호 + 대주 + 동참자` 세 가지 요소가 완벽히 일치해야만 중복으로 간주하도록 강화.

## 5. 자동 발송 스케줄러 기능
- 매일 지정된 시간에 "만료 1개월 전", "2주 전", "1주 전" 대상자에게 자동으로 안내 문자가 발송되는 백그라운드 스케줄러 구현.
- 당일 앱이 꺼져 있어서 시간이 지나갔더라도, 늦게 앱을 켜면 놓친 문자를 스스로 파악해 `1회 한정 즉시 발송` 하도록 보정 로직 적용.
- 로컬 파일(`auto-sms-log.json`) 기록을 통해 `신도번호+대주+동참자` 단위로 동일 조건의 중복 발송을 완벽히 차단.

# 프로젝트 코드 구조 

## 1. 백엔드
*   **`main.cjs`**: 앱이 켜질 때 가장 먼저 실행되는 컨트롤 타워. 화면(창)을 띄우고 아래의 서비스 모듈들을 조립
*   **`preload.cjs`**: 보안을 위해 프론트엔드와 백엔드를 격리하고, 소통할 수 있는 매개체 역할
*   **`services/memberService.cjs`**: 엑셀 업로드 시 중복을 검사하고 회원 데이터를 저장/삭제
*   **`services/smsService.cjs`**: 솔라피(SOLAPI) 서버와 통신하여 실제 문자를 쏘고, 발송 이력을 저장
*   **`services/autoSmsService.cjs`**: 백그라운드에서 매 1분마다 시간을 감시하고, 지정된 시간이 되면 만료 대상자를 추려내어 자동으로 문자를 발송하는 스케줄러

## 2. 프론트엔드
*   **`Dashboard.tsx`**: 앱을 켜면 보이는 메인 화면. 엑셀 회원 목록(표), 검색창, 요약 통계 등
*   **`components/ui/`**: 팝업창과 UI 부품들이 모여있는 폴더
    *   `smsSendModal.tsx` / `smsHistoryModal.tsx`: 문자 발송 창 / 과거 발송 영수증 확인 창
    *   `autoSmsSettingsModal.tsx`: 자동 발송(시간, 조건별 템플릿) 설정 창
    *   `solapiSetupModal.tsx`: 솔라피 연동 API 키 입력 및 사용 가이드 창
    *   `excel-upload-dialog.tsx`: 엑셀 파일을 올렸을 때 유효성을 검증하고 덮어쓰기 여부를 묻는 창
    *   `badge.tsx`, `button.tsx` 등: 이리저리 조립해서 쓰는 레고 블럭(기본 디자인 요소)들
*   **`utils/macroUtils.ts`**: 자동문자나 일반문자에서 쓰는 `{대주}` 같은 '치환 변수'를 동적으로 뽑아내는 등 이리저리 쓰이는 유용한 도구(함수)
*   **`hooks/`**: 데이터를 새로고침하고 저장하는 등의 동작 패턴을 재사용하기 위해 모아둔 곳
